# Finger 插件技术栈推荐：最简且最健壮方案

对于 Burp Suite 插件开发，要达到“最简单”且“最健壮”的平衡，我建议采用以下技术栈：

## 1. 核心技术栈

| 维度 | 推荐方案 | 理由 |
| :--- | :--- | :--- |
| **编程语言** | **Java 17 (LTS)** | Burp Suite 运行在 JVM 上。Java 是最原生、兼容性最好且调试最简单的选择。Java 17 提供了现代语法特性且非常稳定。 |
| **构建工具** | **Maven 3.8+** | 相比 Gradle，Maven 采用 XML 配置，逻辑更线性、更易预测，没有复杂的生命周期和缓存问题，极其健壮。 |
| **Burp API** | **Montoya API** | Burp 最新的官方 API，设计更符合现代编程习惯，提供了比旧版 Extender 更强大的异步处理和 UI 交互能力。 |
| **指纹格式** | **JSON** | 跨语言通用，读写简单，使用 `Jackson` 或 `Gson` 可以极其健壮地处理数据解析。 |

## 2. 关键库选型 (最小化依赖)

为了保持健壮性，我们应尽量减少外部依赖，仅保留核心：

1.  **burp-montoya-api**: 核心 API 支持。
2.  **Gson / Jackson**: 用于高性能、健壮的指纹库解析。
3.  **FlatLaf (可选)**: 如果需要美观的 UI，这是目前 Burp 社区公认最稳定、支持 Dark Mode 的 Swing 皮肤库。
4.  **Google Guava**: 提供极其稳健的并发工具和集合处理（如缓存、速率限制等），这对主动探测的频率控制至关重要。

## 3. 为什么这个方案最健壮？

-   **原生兼容**: Java + Maven 是 Burp 插件开发的“官方路线”，几乎不会遇到类加载器冲突或环境兼容性问题。
-   **易于维护**: 逻辑清晰，新人接手成本极低。
-   **调试友好**: 在 IntelliJ IDEA 中通过 Maven 调试 Burp 插件是目前最成熟的开发流程。
-   **频率控制**: 利用 Guava 的 `RateLimiter` 可以简单而健壮地实现主动探测的速率限制，防止被 WAF 封锁。
-   **高度模块化**: 强制要求代码与数据的解耦。代码库中不应包含大型硬编码数据，所有指纹通过外部模块化文件加载，确保核心逻辑的轻量化。

## 4. 项目结构建议 (模块化)

```text
finger/
├── src/main/java/          # 核心逻辑
├── src/main/resources/
│   └── fingerprints/       # 模块化指纹库目录
│       ├── cms/            # 按类别划分
│       ├── frameworks/     
│       └── middleware/     
├── docs/                   # 文档
└── pom.xml                 # Maven 配置文件
```
